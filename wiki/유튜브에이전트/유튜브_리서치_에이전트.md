# 유튜브 리서치 AI 에이전트

**summary**: 유튜브 채널·검색어를 자동 모니터링하고 영상 자막을 분석해 구조화된 요약(브리프)을 Google Sheets에 저장하는 노코드 AI 에이전트. [[Gumloop]]로 약 10분 만에 구축 가능하며, 좁은 범위에서 시작해 품질 검증 후 점진 확장하는 것이 핵심.

## 개요
유튜브 리서치 에이전트는 지정한 분야의 유튜브 채널이나 검색어를 주기적으로 모니터링하고, 신규 영상의 자막(Transcript)을 읽어 구조화된 요약 브리프를 자동 생성·저장하는 워크플로우다. [[Gumloop]] 같은 노코드 에이전트 빌더로 코딩 없이 구축할 수 있다. 정보 큐레이션·트렌드 추적·리서치 시간 절감에 활용된다.

## 구축 단계 (Gumloop 기준)
1. **에이전트 생성**: Gumloop(gumloop.com/personal/agents) → Create Agent → 이름 지정 (예: Video Brief Builder)
2. **앱 연동**: YouTube(영상 검색·자막 수집) + Google Sheets(결과 저장)
3. **핵심 프롬프트 입력**: 산업/채널/기간/출력항목을 명시한 역할 지시
4. **분석 범위 제한**: 초기엔 분야 1개 + 신뢰 채널 3~5개(또는 검색어 1~2개), 기간 최근 24~48시간
5. **출력 형식 고정**: 제목·URL·핵심 3~5개·중요성·후속 아이디어·유용성 점수(1~5)·What Changed
6. **실행·검토**: 링크 작동·내용 정확성·점수 적절성·노이즈 점검
7. **점진 확장**: 한 번에 하나씩 채널·주제·검색어 추가하며 테스트
8. **스케줄 실행**: 수동보다 매일 같은 시간 자동 실행 권장

## 핵심 프롬프트 템플릿
> "Build me a YouTube monitor for [industry]. Check [channels or search terms], find videos from the last [timeframe], read the transcript, and return a brief with title, URL, 3-5 takeaways, significance, follow-up ideas, usefulness score, and a 'what changed' summary. Log everything to a Google Sheet."

## 운영 원칙
- **좁게 시작 → 검증 → 확장**: 범위를 넓히기 전에 출력 품질을 먼저 검증
- **출력 표준화**: 모든 영상에 동일한 항목 구조를 강제해 비교·축적 용이
- **스케줄 자동화**: 매일 정해진 시간 실행으로 정보 누락 방지

## 시사점 — AI 판단력과의 연결
- 에이전트가 생성한 요약·점수를 그대로 신뢰하지 말고 검증·편집해야 함. 이는 [[AI판단력]]에서 강조하는 '판별과 편집' 역량과 직결된다 (링크 작동·내용 정확성·점수 적절성 점검 단계가 이를 반영)
- '좁게 시작해 직접 만들고 검증하며 확장'하는 방식은 [[카파시학습법]]의 직접 만들기·점진 학습 원칙과도 일맥상통
