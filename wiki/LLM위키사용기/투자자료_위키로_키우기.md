# 내 투자 자료를 '위키'로 키운다는 것

**summary**: 흩어진 투자 자료를 단순 저장이 아닌 LLM 기반 '위키'로 키워 지식을 복리로 축적하는 방법론. 자료 간 교차연결·재방문·점진적 업데이트가 핵심이며, AI가 정리한 내용을 직접 검증·편집하는 태도가 안목을 만든다.

**updated**: 2026-06-03

## 개요
캬오의 일상다반사 블로그 글로, 투자 자료(기사·리포트·메모·캡처)를 단순히 '저장'하는 데 그치지 않고 LLM을 활용해 서로 연결되고 점점 풍부해지는 '위키'로 키워나가는 방법론을 다룬다. 핵심은 자료를 모으는 것이 아니라 자료 간 '연결'과 '재방문'을 통해 지식을 복리처럼 누적하는 것이다.

## 핵심 포인트
- **저장만 하면 죽은 데이터**: 자료를 쌓기만 하면 다시 찾지 못한다. 검색 가능성과 교차참조가 자료를 살아있게 만든다
- **LLM 기반 위키 운영**: 새 소스를 LLM에 주면 기존 페이지를 업데이트하고, 새 개념·종목은 새 페이지로 생성하며, 고유명사는 위키링크로 교차연결
- **지식의 복리**: 위키는 시간이 지날수록 페이지 간 연결이 늘어나며 가치가 복리처럼 누적된다 — [[장기투자철학_나무를심는투자]]의 '나무 심기' 관점과 일맥상통
- **덮어쓰기 대신 업데이트·병기**: 기존 맥락을 보존하기 위해 상충 내용은 '이견'으로 병기하고 점진적으로 업데이트
- **직접 만들며 안목 형성**: 단순 요약 소비가 아니라 직접 구조를 설계하고 검증·편집하는 과정에서 자신만의 판단 기준이 생긴다 — [[카파시학습법]]의 '직접 만들기', [[AI판단력]]의 '판별과 편집'과 직결

## 시사점 — AI 판단력 및 학습법과의 연결
- AI가 생성·정리해준 위키 내용을 그대로 신뢰하지 말고 직접 검증·편집해야 한다 → [[AI판단력]]
- '좁게 시작해 직접 만들고 점진 확장'하는 운영 방식은 [[카파시학습법]] 및 [[유튜브_리서치_에이전트]]의 운영 원칙과 동일한 맥락
- 위키 자동화 자체를 노코드/에이전트로 확장하는 흐름은 [[Gumloop]] 활용 사례와 연결 가능
