# LLM Wiki Schema — 투자 지식베이스

## 목적
이 위키는 투자 관련 자료를 읽고, 지식을 복리로 축적하기 위한 마크다운 기반 지식베이스다.
LLM이 새 소스를 추가할 때마다 기존 페이지를 업데이트하고 교차참조를 유지한다.

---

## 디렉토리 구조

```
sources/          # 원본 자료 (절대 수정하지 않음)
  articles/       # 기사, 리포트, 공시
  earnings/       # 실적발표, 컨퍼런스콜 transcript
  notes/          # 직접 작성한 메모

wiki/             # LLM이 생성·관리하는 페이지
  companies/      # 종목 페이지 (예: 삼성전자.md, NVDA.md)
  sectors/        # 섹터 페이지 (예: 반도체.md, 배터리.md)
  concepts/       # 투자 개념 (예: PER.md, 금리.md, DCF.md)
  macro/          # 거시경제 (예: 연준.md, 환율.md, 인플레이션.md)

schema/           # 운영 규칙 (이 파일)
```

---

## 페이지 포맷

모든 wiki 페이지는 아래 형식을 따른다.

```markdown
# [페이지 제목]

**summary**: 한 줄 요약 (필수 — LLM이 파일 전체를 읽지 않아도 파악 가능하도록)
**tags**: #태그1 #태그2
**updated**: YYYY-MM-DD

---

## 개요
[2~5문장 설명]

## 핵심 포인트
- 포인트 1
- 포인트 2

## 관련 항목
- [[연관 페이지 1]]
- [[연관 페이지 2]]

## 소스
- [소스 제목](../sources/경로)
```

---

## Ingest 규칙 (새 소스 추가 시)

1. `sources/`에 원본 파일을 저장한다 (수정 금지).
2. 소스를 읽고 관련된 wiki 페이지를 **최대 10~15개** 업데이트한다.
3. **새 페이지 생성보다 기존 페이지 업데이트를 우선**한다.
4. 새 개념·종목·섹터가 등장하면 wiki에 새 페이지를 만든다.
5. 모든 고유명사(종목명, 인물, 기관)는 `[[위키링크]]` 형식으로 연결한다.
6. 각 페이지의 `updated` 날짜를 갱신한다.
7. 상충되는 정보가 있으면 기존 내용을 지우지 말고 "이견" 섹션에 병기한다.

**Ingest 프롬프트 예시:**
```
sources/articles/[파일명]을 읽고, schema/SCHEMA.md 규칙에 따라
관련 wiki 페이지들을 업데이트해줘. 새 개념이나 종목이 있으면 페이지를 만들고,
기존 페이지들과 교차참조를 걸어줘.
```

---

## Query 규칙 (질문할 때)

- wiki/ 폴더 전체를 먼저 읽고 답변한다.
- 답변 시 근거가 된 wiki 페이지를 반드시 인용한다.
- 위키에 없는 내용은 "위키에 없음"으로 명시하고 추측하지 않는다.

**Query 프롬프트 예시:**
```
wiki/ 파일들을 읽고, [질문]에 대해 출처 페이지와 함께 답해줘.
```

---

## Lint 규칙 (주기적 유지보수)

아래 문제를 탐지하고 수정 제안을 제시한다:
- 모순된 정보 (같은 수치가 다르게 기재된 경우)
- 고아 페이지 (어디서도 링크되지 않는 페이지)
- 빈 섹션 또는 업데이트가 오래된 페이지
- `[[링크]]`가 걸려 있지만 파일이 존재하지 않는 경우

**Lint 프롬프트 예시:**
```
wiki/ 전체를 스캔해서 모순, 고아 페이지, 깨진 링크, 오래된 내용을 찾아줘.
```
