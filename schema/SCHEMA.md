# LLM Wiki Schema

## 목적
투자 및 부업 관련 지식을 복리로 축적하는 마크다운 기반 위키.
LLM이 새 소스를 추가할 때마다 기존 페이지를 업데이트하고 교차참조를 유지한다.

---

## 디렉토리 구조

```
sources/
  투자/
    articles/       # 투자 기사, 리포트, 공시
    earnings/       # 실적발표, 컨퍼런스콜
    notes/          # 투자 관련 메모
  부업/
    알프랩/          # 알프랩 관련 자료

wiki/
  투자/
    companies/      # 종목 페이지 (삼성전자.md, NVDA.md)
    sectors/        # 섹터 (반도체.md, 배터리.md)
    concepts/       # 투자 개념 (PER.md, DCF.md)
    macro/          # 거시경제 (연준.md, 환율.md)
  부업/
    알프랩/
      시장트렌드/    # 화장품 시장, 트렌드 분석
      브랜드전략/    # 브랜드 포지셔닝, 전략
      마케팅/        # 마케팅, 콘텐츠, SNS
      경쟁사/        # 경쟁 브랜드 분석
      제품개발/      # 원료, 제형, 제품 기획
      유통커머스/    # 채널, 플랫폼, 커머스 전략

schema/
  SCHEMA.md         # 이 파일
```

---

## 페이지 포맷

모든 wiki 페이지는 아래 형식을 따른다.

```markdown
# [페이지 제목]

**summary**: 한 줄 요약
**tags**: #태그1 #태그2
**updated**: YYYY-MM-DD

---

## 개요

## 핵심 포인트
- 포인트 1

## 관련 항목
- [[연관 페이지]]

## 소스
- [소스 제목](../../sources/경로)
```

---

## Ingest 규칙

1. 소스를 `sources/[섹션]/...` 에 저장한다 (수정 금지).
2. 소스를 읽고 관련 wiki 페이지를 **최대 10~15개** 업데이트한다.
3. **새 페이지 생성보다 기존 페이지 업데이트를 우선**한다.
4. 새 개념/종목/브랜드가 등장하면 해당 섹션에 새 페이지를 만든다.
5. 모든 고유명사는 `[[위키링크]]` 형식으로 연결한다.
6. 섹션 간 교차참조도 허용한다 (예: 투자 관점에서 화장품 시장 언급 시 알프랩 페이지 링크).
7. 상충되는 정보는 "이견" 섹션에 병기한다.

**투자 Ingest 프롬프트:**
```
sources/투자/articles/[파일명]을 읽고, schema/SCHEMA.md 규칙에 따라
wiki/투자/ 하위 관련 페이지들을 업데이트해줘.
```

**알프랩 Ingest 프롬프트:**
```
sources/부업/알프랩/[파일명]을 읽고, schema/SCHEMA.md 규칙에 따라
wiki/부업/알프랩/ 하위 관련 페이지들을 업데이트해줘.
```

---

## Query 규칙

```
wiki/[투자 or 부업/알프랩]/ 파일들을 읽고,
[질문]에 대해 출처 페이지와 함께 답해줘.
```

---

## Lint 규칙

```
wiki/ 전체를 스캔해서 모순, 고아 페이지, 깨진 링크, 오래된 내용을 찾아줘.
```
